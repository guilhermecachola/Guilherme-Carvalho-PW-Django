Componentes

  • Model — define a estrutura dos dados e comunica com a base de dados. Cada modelo corresponde a uma tabela.
  • View — recebe o pedido HTTP, consulta os modelos e devolve uma resposta (normalmente um template renderizado).
  • Template — ficheiro HTML que recebe os dados da view e os apresenta ao utilizador.

Fluxo de informação

  • 1. O browser faz um pedido HTTP para uma URL.
  • 2. O Django consulta o urls.py e encaminha para a view correspondente.
  • 3. A view consulta os modelos (base de dados) e recolhe os dados necessários.
  • 4. A view passa os dados ao template HTML.
  • 5. O template é renderizado e devolvido ao browser.
